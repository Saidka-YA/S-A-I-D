# Криптографическое консольное приложение

## Архитектура

Плагинная система: каждый шифр — отдельный `.so`, загружаемый через `dlopen`/`dlsym`.
`crypto_app` знает только об интерфейсе `plugin_run()` из `plugin_api.h`.

```
crypto_app              ← главный бинарник (меню + dlopen)
│
├── libaes.so           ← AES-128 CFB
├── liba51.so           ← A5/1
├── librsa.so           ← RSA  (зависит от libgmp)
├── libhill_vernam.so   ← Хилл + Вернам
├── libbifid_adfgx.so   ← Бифид + ADFGX
├── libaffine.so        ← Аффинный шифр  [НОВЫЙ]
└── libplayfair.so      ← Шифр Плейфера  [НОВЫЙ]
```

Путь к `.so` определяется через `/proc/self/exe` — запускать можно из любой директории.

## Требования

- **ОС**: Ubuntu Linux (20.04 / 22.04 / 24.04)
- **Компилятор**: g++ с поддержкой C++17
- **Библиотека**: libgmp-dev (только для RSA)

---

## Установка зависимостей

```bash
sudo apt update
sudo apt install -y build-essential libgmp-dev
```

---

## Сборка

```bash
cd crypto_project
make
```

После сборки в папке появятся `crypto_app` и семь `.so` файлов.

```bash
make clean   # очистить объектники и бинарники
```

---

## Запуск

```bash
./crypto_app
```

Запускать можно из любого места — `.so` ищутся рядом с `crypto_app` автоматически.

## Описание шифров

### 1 · AES-128 (CFB)
Симметричный блочный шифр. Ключ (16 байт) генерируется автоматически → `aesKey.bin`.

### 2 · A5/1
Поточный шифр на трёх РСЛОС (GSM). Ключ (8 байт) → `a51Key.bin`.

### 3 · RSA
Асимметричный шифр, ключи 1024 бит. Генерация занимает несколько секунд — это норма.
Ключи → `rsaKey.bin`.

### 4 · Хилл / Вернам
Классические символьные шифры, поддержка кириллицы и латиницы.
- **Хилл**: матрица 2–8×2–8, ключевое слово обязательно.
- **Вернам**: ключ = длина текста, одноразовый блокнот.

### 5 · Бифид / ADFGX
Полиграфические шифры.
- **Бифид**: без ключа.
- **ADFGX**: ключевое слово обязательно.

### 6 · Аффинный шифр
Байтовый аффинный шифр: `E(x) = (a·x + b) mod 256`.
- **a** — нечётное число от 1 до 255 (иначе шифр необратим).
- **b** — сдвиг от 0 до 255.
- Ключ: задать вручную или сгенерировать случайно → `affineKey.bin`.

### 7 · Шифр Плейфера
Матрица 16×16 на всех 256 байтах. Ключ минимум 16 байт.
- Задать ключевым словом, hex-строкой или сгенерировать → `playfairKey.bin`.
- Входные данные дополняются до чётной длины нулевым байтом.

## Валидация пользовательского ввода

Все шифры используют общий модуль `src/validation.cpp` / `include/validation.h`.

| Ситуация | Поведение |
|---|---|
| Нечисловой ввод числа | `[!] Ошибка: введены не числовые символы. Введите целое число от N до M.` |
| Число вне диапазона | `[!] Ошибка: число X вне допустимого диапазона [N, M]. Попробуйте снова.` |
| Пустая строка текста | `[!] Ошибка: текст не может быть пустым.` |
| Пустой путь | `[!] Ошибка: путь не может быть пустым.` |
| Файл не существует | `[!] Ошибка: файл 'path' не найден или нет прав на чтение.` |
| Пустой входной файл | `[!] Ошибка: файл 'path' пустой.` |
| Директория не существует | `[!] Ошибка: директория для файла 'path' не существует.` |
| Нет прав на запись | `[!] Ошибка: невозможно создать/открыть файл 'path'.` |
| Чётный параметр `a` аффинного | `[!] Ошибка: параметр 'a' должен быть нечётным (иначе шифр необратим).` |
| Неверная длина hex-ключа | `[!] Ошибка: ожидается ровно N hex-символов (введено M).` |
| Недопустимые hex-символы | `[!] Ошибка: недопустимые символы. Используйте только 0-9, a-f, A-F.` |
| Пустое ключевое слово | `[!] Ошибка: ключевое слово не может быть пустым.` |
| Непечатаемые символы в ключе | `[!] Ошибка: ключевое слово содержит непечатаемые символы.` |
| Повреждённый файл ключа | Описание с указанием файла и причины |
| Неверный формат шифротекста (Хилл) | `[!] Ошибка: первая строка файла шифротекста должна быть числом.` |

После каждой ошибки программа запрашивает ввод повторно — не завершается и не падает.

## Как добавить новый шифр

1. Создать `libs/lib_НОВЫЙ.cpp` с `extern "C" void plugin_run() { ... }`,
   использовать функции из `validation.h` для ввода.
2. Добавить исходники в `src/`.
3. В `Makefile` добавить переменные `НОВЫЙ_SRCS`, `НОВЫЙ_OBJS`, правило `libНОВЫЙ.so`.
4. В `src/main.cpp` добавить строку в `PLUGINS[]` и обновить `PLUGIN_COUNT`.

## Устранение неполадок

| Симптом | Причина | Решение |
|---|---|---|
| `libgmp not found` при сборке | Не установлена GMP | `sudo apt install libgmp-dev` |
| `.so: cannot open shared object` | `.so` не рядом с бинарником | Скопировать все `.so` в папку с `crypto_app` |
| `g++: error: unrecognized` | Старый g++ < 8 | `sudo apt install g++-10`, заменить `CXX = g++-10` |
| `aesKey.bin не найден` | Сначала зашифруйте — ключ создаётся автоматически | — |
| RSA генерация ≈ 10 сек | Норма для 512-битных простых чисел | Дождаться |

# 🔐 Said's Ciphers

Реализация трёх криптографических алгоритмов на C++: **A5/1**, **AES-128 (CFB)** и **RSA**.  
Каждый алгоритм — самостоятельный модуль со своим `main.cpp`, позволяющий шифровать и расшифровывать файлы или текст прямо из консоли.

---

## 📁 Структура репозитория

```
Saids_Ciphers/
├── A51/                        # Потоковый шифр A5/1
│   ├── A51Cipher.cpp/.h        # Реализация РСЛОС и генератора гаммы
│   ├── A51EncryptDecrypt.h     # Функции шифрования/расшифрования
│   ├── A51EncryptDecryptBlock.cpp
│   ├── A51utilities.cpp/.h     # Вспомогательные функции (файлы, ключи, padding)
│   └── A51main.cpp             # Точка входа
│
├── AES Realization/            # Блочный шифр AES-128 в режиме CFB
│   ├── AES128.cpp/.h           # Раундовые операции (SubBytes, ShiftRows, MixColumns...)
│   ├── CFBmode.cpp/.h          # Режим шифрования CFB
│   ├── Utilities.cpp/.h        # Вспомогательные функции
│   └── main.cpp                # Точка входа
│
└── RSA Realization/            # Асимметричный алгоритм RSA
    ├── RSAblock.cpp/.h         # Блочное шифрование/расшифрование
    ├── RSAutilities.cpp/.h     # Генерация простых чисел, работа с ключами (GMP)
    └── main.cpp                # Точка входа
```

## ⚙️ Алгоритмы

### 🔹 A5/1
Потоковый шифр, используемый в стандарте GSM. Реализованы три регистра сдвига с линейной обратной связью (РСЛОС):
- **R1** — 19 бит, полином: x¹⁹ + x¹⁸ + x¹⁷ + x¹⁴ + 1
- **R2** — 22 бита, полином: x²² + x²¹ + 1
- **R3** — 23 бита, полином: x²³ + x²² + x²¹ + x⁸ + 1

Тактирование по принципу большинства, 64-битный ключ + 22-битный номер фрейма. Ключ сохраняется в файл `a51Key.bin`.

### 🔹 AES-128 (CFB)
Блочный шифр с длиной ключа 128 бит в режиме обратной связи по шифртексту (CFB). Реализованы все стандартные раундовые операции: SubBytes, ShiftRows, MixColumns, AddRoundKey, расписание ключей. Ключ сохраняется в файл `aesKey.bin`.

### 🔹 RSA
Асимметричный алгоритм с блочной обработкой данных. Использует библиотеку **GMP** для работы с большими числами. Генерирует пару ключей (публичный + приватный), сохраняет в файл `rsaKey.bin`. Блок данных — 64 байта.

## 🔧 Зависимости

|                     Зависимость                     |    Нужна для    |
|-----------------------------------------------------|-----------------|
| Компилятор C++17 (g++ / clang++)                    | Все модули      |
| [GMP (GNU Multiple Precision)](https://gmplib.org/) | RSA Realization |

Установка GMP:
```bash
# Ubuntu / Debian
sudo apt install libgmp-dev

# Arch Linux
sudo pacman -S gmp

# macOS (Homebrew)
brew install gmp
```

## 🚀 Сборка и запуск

### A5/1
```bash
cd A51
g++ -std=c++17 A51main.cpp A51Cipher.cpp A51EncryptDecryptBlock.cpp A51utilities.cpp -o a51
./a51
```

### AES-128 (CFB)
```bash
cd "AES Realization"
g++ -std=c++17 main.cpp AES128.cpp CFBmode.cpp Utilities.cpp -o aes
./aes
```

### RSA
```bash
cd "RSA Realization"
g++ -std=c++17 main.cpp RSAblock.cpp RSAutilities.cpp -lgmp -o rsa
./rsa
```

## 💻 Использование

После запуска каждой программы появляется меню:

```
1 - Зашифровать файл
2 - Расшифровать файл
3 - Зашифровать текст (консоль)
Выбор:
```

**Режим 1 — шифрование файла:**  
Вводишь путь к входному файлу и путь для сохранения результата. Ключ генерируется автоматически и сохраняется рядом с исполняемым файлом.

**Режим 2 — расшифрование файла:**  
Вводишь путь к зашифрованному файлу. Ключ считывается автоматически из файла (`a51Key.bin` / `aesKey.bin` / `rsaKey.bin`).

**Режим 3 — шифрование текста в консоли:**  
Вводишь текст, получаешь зашифрованный вывод в hex и сразу же расшифрованный результат для проверки.

> ⚠️ Режим консольного ввода не поддерживает многострочный текст. Для длинных данных используй режим работы с файлами.

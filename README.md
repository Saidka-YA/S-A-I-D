# RGR-

Добро пожаловать! В этой ветке представлены консольные интерактивные реализации классических шифров (ADFGX и Бифид), адаптированные под современные реалии работы с данными.
Основная фишка этих реализаций — полная поддержка UTF-8 (включая русский язык, спецсимволы и эмодзи) и защита от «кракозябр» при выводе в консоль.
Это достигнуто за счет интеграции промежуточного Hex-слоя. 
Ключевые особенности проекта:
  1. Универсальность данных: 
    Шифровать можно абсолютно любой текст — от английского и русского до бинарных файлов, так как алгоритмы работают на уровне байтов, а не фиксированных алфавитов.
    Безопасный Hex-интерфейс: Зашифрованный текст выводится в виде понятных печатных символов. Его легко копировать, пересылать и вставлять обратно для дешифровки.
    Гибкий ввод/вывод: Поддерживается как прямая работа в консоли, так и чтение/запись через файлы .txt (в бинарном режиме ios::binary).
    Кроссплатформенный UTF-8: Встроен автоматический фикс кодировки консоли для Windows (65001), чтобы кириллица отображалась корректно из коробки.
  2. Логика работы шифров
       2.1 Шифр ADFGX (Модернизированный)
     Классический немецкий шифр времен Первой мировой войны, переосмысленный через Hex-матрицу.
     Фракционирование (Разбиение): Каждый байт входящего текста превращается в две Hex-цифры (например, буква А в UTF-8 -> D0 90).
     Каждая Hex-цифра проецируется на сетку, где строкам и столбцам соответствуют буквы A, D, F, G, X.
     Один исходный символ превращается в 4 буквы шифра.
     Столбцовая перестановка: Полученная строка ADFGX записывается в таблицу, ширина которой равна длине вашего ключевого слова.
     Затем столбцы таблицы сортируются по алфавитному порядку букв ключа, и текст считывается вертикально.
       2.2 Шифр Бифида (Bifid Cipher)
     Двухбуквенный шифр, сочетающий в себе подстановку и перестановку (диффузию) элементов.
     Координатная сетка: Текст также переводится в Hex-последовательность. Каждому Hex-символу присваиваются координаты строки и столбца в матрице.
     Перемешивание: Координаты строк всех символов выписываются в один ряд, а координаты столбцов — строго за ними.
     Полученный единый массив координат снова режется попарно.
     Сборка: По этим новым парам координат из матрицы достаются новые Hex-символы.
     На выходе получается монолитная зашифрованная Hex-строка, полностью скрывающая структуру исходного языка.
     
Инструкция по сборке и запуску:
Для компиляции программы вам понадобится любой современный компилятор C++ с поддержкой стандарта C++11 или выше (например, GCC/MinGW, Clang, MSVC).
Компиляция через терминал (g++):
Bash
  g++ -std=c++11 main.cpp -o ciphers_app
  Запуск:
  ./ciphers_app

# Криптографический инструмент: Шифр Хилла и Шифр Вернама (по модулю m)

Добро пожаловать! В этой ветке представлены консольные интерактивные реализации двух классических шифров — **Хилла** и **Вернама (аддитивный вариант)**.

## Ключевые особенности проекта

- **Конкретность данных**:  
  Шифровать можно любой текст — от английского до русского, так как алгоритмы работают с символами ASCII и UTF8.

- **Гибкий ввод/вывод**:  
  Поддерживается как прямая работа в консоли (ввод текста), так и чтение/запись через файлы.

## Логика работы шифров

### 1. Шифр Вернама (аддитивный по модулю m)

Классический поточный шифр, в котором гамма добавляется к открытому тексту по модулю заданного числа **m**. При истинной случайности ключа и его длине, равной длине сообщения, даёт абсолютную стойкость («одноразовый блокнот»).

- **Преобразование**:  
  Исходный текст переводится в последовательность байт. Ключ (строка) также преобразуется в байты и при необходимости циклически повторяется до длины текста.  
  Для каждого байта открытого текста `P` и соответствующего байта ключа `K` вычисляется зашифрованный байт `C = (P + K) mod m`.  
  При дешифровании: `P = (C - K) mod m`.  
  По умолчанию `m = 256`.

- **Выходные данные**:  
  Результат `C` — это байты, которые могут быть непечатными. Поэтому программа автоматически кодирует шифротекст в **Hex-строку** (символы `0-9A-F`). При дешифровке Hex преобразуется обратно в байты, и вычитание ключа восстанавливает исходный текст.

- **Важное замечание**:  
  Для криптостойкости в реальных задачах ключ должен быть случайным, длиной не менее сообщения и использоваться один раз. В демонстрационных целях допускается короткий ключ (циклическое повторение), но это снижает защиту.

### 2. Шифр Хилла (матричный блочный шифр)

Классический шифр, основанный на линейной алгебре. Он оперирует блоками символов одинаковой длины, преобразуя их с помощью квадратной матрицы (ключа).

- **Подготовка данных**:  
  Исходный текст переводится в байтовый массив. Каждый байт интерпретируется как число от 0 до 255. Ключ — это квадратная матрица `n x n` (например, 3×3), заданная пользователем. Матрица должна быть обратимой по модулю 256.

- **Шифрование**:  
  Байты разбиваются на блоки по `n`. Каждый блок умножается на матрицу ключа по модулю 256, в результате получается новый блок зашифрованных байтов. Если длина сообщения не кратна размеру блока, добавляется случайный или нулевой паддинг.

- **Выходные данные**:  
  Так как результат может содержать непечатные байты, зашифрованный поток кодируется в **Hex-строку** аналогично шифру Вернама. При дешифровке Hex восстанавливается в байты, затем используется обратная матрица (по модулю 256) для восстановления исходного текста.

- **Требование к ключу**:  
  Матрица должна быть обратимой. Программа проверяет, что определитель матрицы нечетен (для модуля 256 достаточно нечётности) и что обратная матрица существует. В противном случае будет выведена ошибка.
